# Task H 设计文档（编辑器体验增强）

## 1. 目标

在现有 Task G 基础上完成三项增强：

1. 将 `textarea` 升级为 CodeMirror 6，提供行号与语法高亮。
2. 增加未保存保护：浏览器离开提示 + 应用内切换提示。
3. 将重命名交互从 `prompt` 升级为可即时校验的输入弹窗，并加冲突提示。

## 2. 方案选择

### 方案 A：最小改动（继续堆在 App.vue）
- 优点：快。
- 缺点：状态与交互继续集中，后续维护成本高。

### 方案 B：中等重构（组件+工具拆分）
- 优点：结构清晰。
- 缺点：需要一定文件拆分。

### 方案 C：完整交互升级（已选）
- 在方案 B 基础上，增加未保存三分支决策弹窗（保存并继续/放弃修改/取消）。
- 兼顾可维护性与交互完整度。

已选：**方案 C**。

## 3. 详细设计

### 3.1 CodeMirror 编辑器

- 新增 `CodeEditor.vue`，封装 CodeMirror 实例。
- 支持：
  - 行号
  - 按文件语言选择语法扩展（ts/js/json/markdown/vue）。
  - `Cmd/Ctrl + S` 快捷键触发保存。
- App 只负责传入 `modelValue/language` 与保存回调。

### 3.2 未保存保护

- 新增 `useUnsavedGuard` composable：
  - 管理未保存弹窗状态。
  - 提供异步决策接口（save/discard/cancel）。
  - 挂载 `beforeunload` 浏览器离开提示。
- 应用内切换入口全部走统一 guard：
  - 返回 Library
  - 切换工作区
  - 切换文件
- 若选择“保存并继续”：先调用 `saveCurrentFile()`；保存失败则中断切换。

### 3.3 重命名即时校验

- 新增 `RenameDialog.vue`。
- 新增 `rename-validation.ts` 工具函数。
- 规则（严格）：
  - 非空
  - 不能有首尾空格
  - 不能包含 `/`
  - 不能为 `.` 或 `..`
  - 长度不超过 120
  - 同级目录名冲突禁止提交
- `FileTree` 发起重命名时仅触发事件，App 打开弹窗并进行校验与提交。

### 3.4 交互文案

- 未保存弹窗文案明确说明风险。
- 重命名错误在输入框下方即时显示，确认按钮按校验状态禁用。

## 4. 测试策略

- 工具层：`rename-validation` 单测覆盖所有规则分支。
- App 交互层：
  - 未保存三分支（取消/放弃/保存并继续）。
  - 切换文件场景的未保存拦截。
- 现有测试继续回归，确保 Task G 主链路不回退。

## 5. 非目标

- 不引入多标签编辑。
- 不实现复杂冲突合并。
- 不改动后端 API 语义。
